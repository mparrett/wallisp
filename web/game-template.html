Same host-driven loop as harness/game.mjs, in the browser: a keypress pokes the action into the module's input slots, rerun() re-executes the once-compiled (tick), and the frame it renders via (display …) is written to xterm.js. No SharedArrayBuffer, no cross-origin isolation — nothing blocks inside the wasm.
