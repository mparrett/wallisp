The key type is `SVal`: every subexpression resolves either to a comptime value (`SV_INT`, `SV_TYPE`) or to a residual handle (`SV_RES`, an IR temp). That tag *is* the binding-time lattice. `spec()` then does eval-or-emit per node: comptime arithmetic folds, a runtime load emits an instruction and hands back a temp. `as_temp()` is the comptime→runtime boundary — it materializes a known constant into a `const` only at the moment a residual op actually needs it.

The four behaviors from the design map straight onto lines you can point at:

- **Type as a comptime value.** `A_TYPE` produces an `SV_TYPE`; it flows through `A_TBIN`'s `type_of` slot and picks `mul.i` vs `mul.f`. That's why `dot$i32$n4` emits integer ops and `dot$f32$n3` emits float ops from *identical* source.
- **Unrolling.** `A_REPEAT` reads its count via `spec()`, hits the `barrier()` (count must be comptime), then just runs the body that many times — so `n=4` straight-lines into 4 terms and `n=8` into 8. No loop survives.
- **Identity elision.** The accumulator starts as comptime `0`, so the first `0 + x` is dropped at compile time. That's why `n=4` is 16 instructions, not 17 (8 loads, 4 muls, 3 adds, 1 ret) — comptime knowledge bought you a free strength reduction.
- **Monomorphization.** `(i32, 4)` is requested twice; the cache returns the same function and reports `1 cache hit`. Only 3 functions are actually emitted (the driver re-prints the cached one, which is why you see `dot$i32$n4` twice in the output).

And the thing you were after: the metaprogram never looks at syntax. `main` builds the `body` AST once, as a private compiler struct, and `spec()` only ever reads *values and types* out of `env` — so there's no homoiconicity anywhere, and the residual `IRFunc`s contain no `SV_TYPE` values, no `repeat`, no comptime calls. Each one lowers directly: `IRFunc` → a wasm function, `load … +k` → a constant-offset load, `mul.i/mul.f` → `i32.mul`/`f32.mul`. Nothing comptime ships in the module.

To take it from toy to real, the pieces you'd add are: specializing *call sites* so generic functions can call other generics (thread the same `spec()` through a `A_CALL` that instantiates the callee), an **eval quota** in `A_REPEAT`/recursion since comptime is Turing-complete and can hang the compiler, a real hash for the cache key instead of the linear scan, and an SSA/dominance story for the residual when a loop's count *isn't* comptime (then you emit an actual `loop` block instead of unrolling). But the spine — `SVal`, `spec()`, `as_temp`, `barrier` — stays exactly this shape.
